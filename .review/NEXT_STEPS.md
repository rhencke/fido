2. THE SPECIFICATION-SHAPED FILE ROOT (Master Plan 3.1–3.4)
===============================================================================

Keep the file's SOURCE structure separate from its compilation-unit placement metadata (the path).

  Record GoSourceFile := {
    source_package : PackageClauseSyntax;
    source_imports : list ImportSpecSyntax;
    source_decls   : list TopLevelDeclSyntax
  }.

  Record GoFileNode := {
    file_path   : FilePath;
    file_source : GoSourceFile
  }.

The path belongs to the file-root program node; it is NOT a child production inside Go's source grammar.

2.1 Current supported subgrammar (Master Plan 3.2) — model the PERMANENT source-file categories, but with
intentionally narrow LIVE domains; do NOT make a future source production representable before its semantics
(that repeats the subset-filter mistake):
- package clause: only the currently supported canonical `package main` form;
- imports: absent / intrinsically empty (NOT "representable then rejected wholesale");
- top-level declarations: only the current `func main()` declaration form;
- statements/expressions: the current admitted forms.

Do NOT broaden package/import semantics.  Do NOT represent arbitrary package names and then reject non-main.
Do NOT represent imports and then reject them.

2.2 Path-keyed file set (Master Plan 3.3) — replace the public `dict[path, AST]` with:

  Record GoFileSet := {
    file_members      : list GoFileNode;
    file_paths_unique : NoDup (map file_path file_members)
  }.

Semantically `set[GoFileNode]` with uniqueness by `file_path`.  Required meaning:
- same path twice: impossible;
- different paths with identical source: allowed;
- same path with different source: impossible;
- physical list order: NOT program semantics.

2.3 Program root (Master Plan 3.4):

  Record GoProgram := { prog_module : ModuleSpec ; prog_files : GoFileSet }.

`ModuleSpec` remains OUTSIDE source files (it renders the module's `go.mod`); `go.mod` is not a `FilePath`.
No synthetic source-level program AST node is required.
