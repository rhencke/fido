1.1 Source authority

There is one raw AST:

  GoProgram
    prog_module : ModuleSpec
    prog_files  : GoFileMap

  GoFileMap
    = standard FileMap FilePath GoSourceFile

A file-map binding:

  FilePath -> GoSourceFile

is the file-root program occurrence.

The map key is the sole stored file-path authority.

`GoFileNode` is only:
- duplicate-preserving construction input;
- a derived view of a binding.

It is not the stored map value.

1.2 Source / semantics split

`GoAST` owns source syntax.

`GoIndex` owns structural occurrence identity and navigation.

`GoTypes` owns semantic type/constant relations.

`GoCompile` owns whole-program validity and later resolution/facts/diagnostics.

`GoRender` renders the original source syntax.

Do not:
- type the AST;
- copy the AST;
- rewrite syntax into semantic nodes;
- put compiler facts into syntax;
- put source paths/locations into semantic types.

1.3 Collection authority

Use:
- `Collections.FileMapBase` (`FMapAVL`) for FilePath-keyed maps;
- `Collections.NodeMapBase` (`FMapPositive.PositiveMap`) for local positive IDs.

A thin domain wrapper is allowed.
A Fido-authored collection storage implementation is forbidden.

Do not:
- restore the old association-list map;
- write a trie;
- write a dense primitive array;
- use `PArray` / `Uint63`;
- use list + NoDup as identity-keyed storage;
- add a hidden file slot;
- add a parallel path map;
- use standard `add` in a way that erases duplicate source evidence.

1.4 Occurrence representation

Frozen choices:

- local occurrence IDs are `positive`;
- file root is exactly `1%positive`;
- IDs are assigned by one deterministic file-local preorder;
- NodeKey public identity is `(FilePath, positive)`;
- NodeMeta contains only:
    kind
    parent local ID
    role in parent
    subtree_end local ID
- containing file is carried by the reference;
- source recovery is an occasional table-free source traversal, not a copied syntax cache;
- direct children are enumerated by interval jumps over subtrees;
- ancestor tests use preorder intervals;
- snapshot-local references are indexed by the exact `GoProgram`, not by free-standing index data.

1.5 Accepted C0/C0A/C0B theorem root

Production C2 must preserve and generalize the accepted spike result:

  source syntax
       |
       +-- independent table-free source-occurrence specification
       |
       +-- efficient PositiveMap builder
                  |
                  +-- universal theorem:
                      stored metadata at local ID

                      metadata of exact source occurrence at local ID

A structurally coherent but semantically mislabeled table must be unprovable.

4. CURRENT OCCURRENCE UNIVERSE
===============================================================================

Define one current `SyntaxKind` authority.

Use names compatible with the permanent architecture. At minimum represent:

  KFile
  KPackageClause
  KTopLevelDecl
  KStatement
  KExpression

Do not add kinds for unsupported future syntax.

Do not index:
- list containers;
- punctuation;
- whitespace;
- generated keywords;
- synthetic package objects;
- map bindings as extra AST nodes;
- semantic constants/types;
- DecimalFloat/DecimalComplex components as separate expression occurrences when the AST stores them as fields
  of one literal expression.

Current source occurrences:

4.1 File

Every `FilePath -> GoSourceFile` binding has:
- one file-root occurrence;
- local ID `1`;
- no parent;
- role `RFileRoot`.

4.2 Package clause

Every file has exactly one package-clause occurrence:
- child of file root;
- before declarations in preorder/source order;
- role equivalent to `RFilePackage`.

`PackageClauseSyntax` currently has only `PkgMain`, but the occurrence is real source syntax and must be indexed.

4.3 Imports

`ImportSpecSyntax` is empty.

No import occurrence exists today.

The indexer must structurally consume the category so adding the first import constructor later forces the index
definition/proofs to change. Do not silently ignore a future live import list.

4.4 Top-level declarations

Each `source_decls` entry is one top-level-declaration occurrence:
- in list/source order;
- child of file root;
- role equivalent to `RFileDecl index`.

Today each value is `DMain body`.

4.5 Statements

Each `DMain` body statement is one statement occurrence:
- in source order;
- child of the declaration;
- role equivalent to `RDeclStmt index`.

Today each value is `SPrintln args`.

4.6 Statement expressions

Each `SPrintln` argument is one expression occurrence:
- in source order;
- child of the statement;
- role equivalent to `RPrintlnArg index`.

Do not collapse structurally equal arguments.

4.7 Expression children

Leaf expression constructors have no indexed children:

  EBool
  EInt
  ENeg
  EString
  EFloat
  EComplex

`ENeg` carries an `N`; it is not currently a generic unary-expression node with a child.

`EComplex` carries one `DecimalComplex`; its components are not separate `GoExpr` children in the current AST.

Each explicit conversion has exactly one expression child:

  EIntConvert target operand
  EFloatConvert target operand
  EComplexConvert target operand

The operand:
- is the next preorder occurrence;
- is a child of the conversion expression;
- has role equivalent to `RConversionOperand`.

One role is sufficient unless a genuinely different current structural relationship requires distinction. Do
not create redundant roles for integer/float/complex conversion merely because their constructors differ.

4.8 Canonical preorder

For each file:

  file root
  package clause
  declarations in source order
    declaration
    statements in source order
      statement
      println arguments in source order
        expression preorder
          conversion operand recursively

The first local ID is `1`.

Every subsequent indexed occurrence receives the next positive integer.

No ID depends on:
- syntax equality;
- hash-consing;
- map shape;
- file insertion order;
- semantic validity;
- compiler acceptance;
- literal payload equality.

- never consults NodeTable;

- never consults index_file;
